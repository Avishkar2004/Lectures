Amazon EC2 Autoscaling :-

Autoscaling is region specific

- It allows you to automatically adjust the number of instances in your application based on demand. 

creating group of EC2 instances that can scale up or down depending on conditions you set
- enables elasticity by scalling horizontally though adding or termination EC2 instances
- Autoscaling ensures that you have the right number of AWS EC2 instances for your needs all the time
- Autoscaling helps you save cost by cutting down the number of instances when not needed, and scalling out to add more instances only when it is required

scale out/ up = instances badana
scale in  = instances ghatana

 Autoscaling Components

1. Launch configuration :-
like instances type, AMI, keypair, security group

2. Autoscaling group :-
group name, group size min(3) max(20), vpc, subnet, health chekc periods 

3. Scaling policy :-
Target value, metric type (cpu utilization)
